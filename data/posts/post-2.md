# Prompt Engineering 心得：让 AI 成为你的编程搭档

> Code is cheap. Show me the prompt.

在日常研究和开发中，Prompt Engineering（提示词工程）已经成为我工作流中不可或缺的一部分。本文分享一些实践心得。

## 什么是好的 Prompt？

一个好的 Prompt 应该具备以下特征：

- **明确性**：清楚表达你想要什么
- **上下文**：提供足够的背景信息
- **约束条件**：指定输出格式和限制
- **示例**：给出期望输出的样例

## 代码生成场景

### 基础模式

```
请用 Python 实现一个函数，功能是 [具体描述]。
要求：
- 使用 [特定库]
- 输入参数为 [参数描述]
- 返回值为 [返回值描述]
- 包含错误处理
- 添加类型注解
```

### 进阶模式：角色设定 + 约束

```
你是一位资深的嵌入式软件工程师，精通 STM32 和 C 语言。
请为 STM32H743 编写一个 [功能描述] 的驱动程序。

约束条件：
- 符合 MISRA C 编码规范
- 最小化动态内存分配
- 包含详细的注释
- 提供使用示例
```

## 论文写作辅助

Prompt 工程在学术写作中也非常有用：

1. **润色**：「请将以下段落改写为学术论文风格，保持技术准确性」
2. **结构化**：「请为以下实验结果生成分析段落，包含数据对比和结论」
3. **审查**：「请检查以下段落的逻辑一致性和语法错误」

## 关键原则

| 原则 | 说明 |
|------|------|
| 迭代优化 | 不要期望一次就得到完美结果，逐步优化 Prompt |
| 分解任务 | 将复杂任务拆分为多个小步骤 |
| 保持怀疑 | 始终验证 AI 生成的代码和信息 |
| 学习模式 | 观察什么样的 Prompt 产生好结果，积累经验 |

## 总结

Prompt Engineering 不是简单的「问问题」，而是一种与 AI 高效协作的技能。掌握好这项技能，能够显著提升你的研发效率。

记住：**Code is cheap. Show me the prompt.** 在 AI 时代，描述问题的能力可能比写代码本身更重要。
